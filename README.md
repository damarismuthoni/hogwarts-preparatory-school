# Hogwarts Preparatory Project Documentation

## Introduction

Welcome to the documentation for the **Hogwarts Preparatory** project! This document provides an overview of the project, its goals, technologies used, and how to get started.

### Project Overview

**Hogwarts Preparatory** is a web application developed as part of an assessment for a frontend developer position. The main objective of the project was to integrate API endpoints into the frontend and display Harry Potter characters. The application also includes additional features such as searching and pagination of results.

## Technologies Used

The project was developed using the following technologies:

- **Next.js**: A popular React framework for building server-rendered React applications.
- **TypeScript**: A statically-typed superset of JavaScript that enhances code quality and maintainability.
- **Cypress**: A testing framework for end-to-end testing of web applications.
- **GitHub**: A platform for hosting and collaborating on software development projects.

## Getting Started

Follow these steps to get the project up and running:

1. Clone the GitHub repository: [GitHub Repository Link](https://github.com/damarismuthoni/hogwarts-preparatory-school)
2. Navigate to the project directory: `cd hogwarts-preparatory`
3. Install dependencies: `npm install`
4. Start the development server: `npm run dev`

## Project Features

### API Integration and Character Display

The core functionality of the project involves integrating API endpoints to fetch Harry Potter character data and displaying it on the frontend. Characters are presented in an organized manner with relevant information.

### Search Functionality

The application includes a search feature that allows users to search for specific Harry Potter characters by name or other attributes. This enhances user experience by enabling quick access to desired information.

### Pagination

To manage a potentially large number of characters, the application implements pagination. This divides the character list into manageable pages, enhancing performance and user-friendliness.
